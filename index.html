Требования к вёрстке макетов:

  1.  Общие требования:

1.          Только блочная верстка (табличную верстку можно использовать для таблиц только по предварительному согласованию с заказчиком).

2.   HTML5/CSS3 верстка (по максимуму использовать теги html5, такие как: main, header, `, section, article, blockquote, abbr, address, aside, figure, figcaption, nav, menu...).

3.          Использование jQuery (все функции, инициализация плагинов выносятся в отдельный файл - custom.js).

4.          Цвет фона для body задан всегда. Если конкретный цвет не указан в макете, использовать белый (#fff).

5.          Размеры шрифтов указывать в относительных единицах измерения - em. Считать относительно основного размера прописанного для body в px.

Пример:

body {

font-size:13px!important;

}

p {

font-size:1em;line-height:1em;

}

Просчитываются значения на сайте http://pxtoem.com/

6.          Основной шрифт, который используется на сайте, нужно прописывать для body с указанием альтернативных вариантов шрифта и типа семейства в конце, например, font-family:'GothamPro',Arial,Tahoma,sans-serif;). Таким образом при написании классов основной шрифт уже не прописывается, а указывается только размер (font-size), межстрочное расстояние (line-height - прописывается для всех текстов, даже если в макете указано auto), цвет (color), жирность (font-weight)...

Пример:

.tel-number{

                  font-size:1em;

                  line-height:1.1em;

                  color:#000000;

                  font-weight:bold;

   }

7.          Вначале style.css должны быть прописаны reset  стили http://prntscr.com/btyc4o (в шаблоне для быстрого старта проекта (Project Template) все нужные стили будут уже собраны).

8.          Стили текстовых параграфов (р) , заголовков (h1-h6) должны быть прописаны по умолчанию - без каких либо классов, если нужно переопределить - наследованием через внешний блок или добавлением класса - модификатора http://prntscr.com/btyepk

9.          Вёрстка должна корректно работать в IE 11+, и последней версии популярных браузеров - Firefox, Opera, Mac Safari, Chrome, iPad, iPhone, Android. Тестируется на сервере https://www.browserstack.com

10. Если используются CSS хаки - нужны комментарии, что это и для какого браузера, а лучше использовать browser_hack.js http://prntscr.com/buvzza 

11. Названия классов и id должны по смыслу соответствовать  применению — например .box-nav, .box-news, .box-footer и т.д…

12. В коде должны быть комментарии основных html блоков:

      <!-- BEGIN FOOTER -->

      <!-- END FOOTER -->

             ...

             CSS файл так же должен быть разбит с помощью строк с комментариями

             /* reset styles */

             /* header */

             /* content */

             /* footer */

               …

13. Страница обязательно должна помещаться без горизонтальных скроллбаров в развернутое на весь экран окно браузера при УКАЗАННОЙ В ЗАДАНИИ горизонтальной составляющей разрешения экрана.

14. Названия картинок должны соответствовать назначению блока на странице или дублировать название блока для которого используется. Например, img_new_1, img_new..., img_product_1, img_product_…        

15. Title (заголовок в шапке браузера) страницы должен соответствовать шаблону “Название сайта – Название страницы”, например “Smileserver.com - Home page”

16. HTML-макет, CSS и JS файлы должны быть в кодировке UTF8.

17. Необходимо оптимизировать изображения без потери качества на сервисе https://tinypng.com/ или https://kraken.io/.

18. Для иконок, ховеров используются css-спрайты. Если есть svg картинки - на сервере https://icomoon.io/app/#/select они собираются  в шрифт и подключаются.

   !!!Все svg, которые используются в шрифте - обязательно собираем в отдельную папку, которую скидываем вместе с остальными исходниками проекта. Делается это для того, чтобы в любой момент шрифт можно было пересобрать, если нужно добавить что-то, и при этом все иконки остались на местах.

19. Если нужна ретина (уточняется дополнительно в требованиях к верстке проекта) -  используются либо свг картинки либо x2 увеличенные картинки, порезанные из дизайна.

20. Все тексты должны быть реализованы именно в виде текста, а не изображения! Любое отступление от этого правила оговаривается отдельно.

21. Все фоны, градиенты, скругления, тени - по  максимуму что можно делать без использования изображений - на CSS3.

22. Абсолютное позиционирование (position:absolute) использовать по минимуму - только там, где это необходимо.

23.Все блоки, высоту которых ничего в дизайне не мешает сделать динамической, должны иметь именно динамическую (т. е. зависимую от содержания) высоту, а иногда, чтобы ничего не могло потенциально поломать дизайн, нужно задавать и минимальную высоту.

24.Футер должен быть прижат к низу браузера при любом объёме контента. Это делается флексами (в Project Template уже реализовано).

25.Макеты должны проходить 100% html-валидацию на https://validator.w3.org/ . Тут же проверяется валидность CSS файлов https://jigsaw.w3.org/css-validator/ .

28. Скрипты должны цепляться на классы с префиксом .js- либо, если это необходимо -  на #id .

        Следует использовать:

40. Перед сдачей проекта еще раз внимательно изучить и сверить свою работу на соответствие стандартам

 

2. Структура кода:

 

1.Вся страница должна быть заключена в главный тег <div> с классом  .main-wrapper. Этот класс отвечает за минимальную ширину сайта и предотвращает появление горизонтального скролла до достижения минимальной ширины за счет свойства overflow: hidden:

 

   .main-wrapper{

             padding: 0 0 0 0;

             min-width: 320px;

             position: relative;

             overflow: hidden;

             min-height:100%;

   }

   Для body и html  в reset стилях прописывается height:100%, поэтому для внешнего дива .main-wrapper ставится min-height:100%; и position: relative; за счет чего минимальная высота блока всегда 100%.

2. Основная структура сайта внутри <div class="main-wrapper">... </div> такая:

 

<!-- BEGIN CONTENT -->

   <main class="content">

             ...                                    

   </main>                          

<!-- CONTENT EOF   →

<!-- BEGIN HEADER -->

   <header>
            ...

   </header>  

<!-- HEADER EOF   -->

<!-- BEGIN FOOTER -->  

   <footer>

             ...             

   </footer>  

<!-- FOOTER EOF  -->

 

3. Контейнер, который центрирует содержимое и задает максимальную ширину сайту - .wrapper, т.е. <div class="wrapper">... </div>

4. Фоны всегда тянем по ширине монитора. К примеру в контенте два блока с разными фонами будут прописываться так:

   <main class="content">

                             <section class="box-contact">

                             </section>                                 data

   </main>

   Всем блокам, у которых фоны растягиваются по ширине монитора, указывается минимальная ширина сайта (min-width)!

5.После дива .main-wrapper добавляется лоадер - иконка, которая будет показываться до загрузки страницы

   <div class="icon-load"></div>

   Соответственно для body добавляется класс  .loaded (<body class="loaded">) через который показывается эта иконка и скрывается див с кодом страницы.

   В style.css это будет прописываться так:

   .loaded .main-wrapper{

                        visibility:hidden;

   }

   .icon-load{

                        background:url(../img/loader.gif) no-repeat left top;

                        width:40px;height:40px;

                        position:fixed;

                        left:50%;top:50%;

                        margin-left:-20px;margin-left:-20px;

                        display:none;

   }

   .loaded .icon-load{

                        display:block;

   }

 

   В custom.js после загрузки страницы удаляем класс loaded:

   $(window).load(function(){

                        $('body').removeClass('loaded');

   });

5. Все плагины подключаются снизу страницы, перед </body>.

scr3. Наименование классов

Для наименования классов используется методология БЭМ: (сперва идём по ссылке и разбираемся что такое БЭМ, после читаем комментарии ниже)

1. Имена блоков пишем через дефис, например .header-search, .main-nav, .box-main-article, .main-article…

2. Элементы блоков прописываются через двойное подчеркивание, например,  .header-search__field, .main-nav__item, .main-article__item

3. Модификаторы элемента - дописываются через одинарное подчеркивание,  если нужно изменить какие-то свойства уже существующего класса. Могут дописываться как самим блокам, так и элементам блока, например .main-nav__item_special, .header-search_pull-left.

Комментарии по второму тестовому

все включая наименования, какие плагины используем, классы которые прописаны, должны и в тестовом так же называться, к примеру та же форма, остальное по аналогии.

По основным моментам тестового:

все иконки есть в свг, их собираем в шрифт как описано в стандарте.

1. в стандарте все включая наименования, какие плагины используем, классы которые прописаны, должны и в тестовом так же называться, к примеру та же форма, остальное по аналогии.

2. по основным моментам тестового, все должно резиниться, и форма тоже, на 1920, поля тянуться по ширине, у лейбла фиксированная ширина вот пример http://prntscr.com/ejjjvj
